Systemctl — это утилита командной строки для управления системой и службами в Linux-системах, использующих systemd в качестве системы инициализации. Systemd — это init-система, которая управляет запуском процессов при загрузке системы и контролирует их работу в дальнейшем. Systemctl позволяет администрировать службы, точки монтирования, устройства и другие системные ресурсы, называемые юнитами.  

Установка systemctl
В большинстве дистрибутивов Linux systemctl входит в пакет systemd и устанавливается по умолчанию. 
Если система использует systemd, но systemctl отсутствует, возможно, пакет повреждён или не установлен полностью. 

Дистрибутив	                Команда
Debian/Ubuntu	                sudo apt update, затем sudo apt install systemd
CentOS, AlmaLinux (RPM-based)	sudo yum update, затем sudo yum install systemd 
Fedora (с DNF)	                sudo dnf upgrade, затем sudo dnf install systemd 

Проверить установилось ли systemctl
systemctl --version

Результат должен быть по аналогии с этим:
systemd 252 (252.39-1~deb12u1)
+PAM +AUDIT +SELINUX +APPARMOR +IMA +SMACK +SECCOMP +GCRYPT -GNUTLS +OPENSSL +ACL +BLKID +CURL +ELFUTILS +FIDO2 +IDN2 -IDN +IPTC +KMOD +LIBCRYPTSETUP +LIBFDISK +PCRE2 -PWQUALITY +P11KIT +QRENCODE +TPM2 +BZIP2 +LZ4 +XZ +ZLIB +ZSTD -BPF_FRAMEWORK -XKBCOMMON +UTMP +SYSVINIT default-hierarchy=unified

где
    systemd 252 (252.39-1~deb12u1) — версия systemd и номер пакета в репозитории Debian 12 (Bookworm);
    +PAM, +AUDIT и т. д. — список включенных при компиляции функций (поддержки библиотек и технологий);
    -GNUTLS — отключённая функция (в данном случае библиотека GNUTLS);
    default-hierarchy=unified — используется унифицированная иерархия cgroups (современный стандарт для Linux).
